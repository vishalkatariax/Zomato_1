# Free Deployment Guide for Restaurant Recommendation System

## Overview
This guide shows how to deploy the complete restaurant recommendation system using free tools and services.

## Free Deployment Options

### 1. Vercel (Frontend)
- **Free Tier**: 100GB bandwidth/month, unlimited sites
- **Best for**: Static HTML/JS, React, Vue
- **Setup**: GitHub integration + automatic deployments

### 2. Railway (Backend)
- **Free Tier**: $5 credit/month (enough for small projects/APIs)
- **Best for**: Docker applications, Python Flask/FastAPI
- **Setup**: GitHub integration + Nixpacks automatic builds

## Recommended Architecture

```
Frontend (Vercel) → API Requests → Backend (Railway)
```

## Step-by-Step Deployment

### Phase 1: Frontend Deployment (Vercel)

1. The frontend consists of the static `index.html` at the root of the repository.
2. Vercel's configuration (`vercel.json`) is set up to serve this directory statically.
3. Push your repository to GitHub.
4. Log into [Vercel](https://vercel.com/) and click "Add New Project".
5. Import your GitHub repository. Vercel will automatically detect the static configuration.
6. Deploy the project. Note the URL Vercel gives you.

### Phase 2: Backend Deployment (Railway)

1. Ensure your backend files (`src/restaurant_app.py`, `requirements.txt`, `Procfile`) are in the repository. The `Procfile` tells Railway how to start the app: `web: gunicorn --chdir src restaurant_app:app`.
2. Log into [Railway](https://railway.app/) and create a "New Project".
3. Select "Deploy from GitHub repo" and choose this repository.
4. Railway will automatically detect the Python environment and build it.
5. Go to the project's **Variables** tab in Railway and add:
   - `GROQ_API_KEY`: Your Groq API key
   - `FLASK_ENV`: `production`
6. Go to the **Settings** tab and click "Generate Domain" to get a public URL for your API (e.g., `https://zomato-ai-backend-production.up.railway.app`).

### Phase 3: Connect Frontend to Backend

1. In your local repository, open `index.html`.
2. Locate the `API_BASE` variable (around line 385):
   ```javascript
   const API_BASE = window.location.hostname === 'localhost' || window.location.hostname === '127.0.0.1'
     ? 'http://localhost:5050'
     : 'https://zomato-ai-backend-production.up.railway.app'; // Replace with actual Railway URL
   ```
3. Replace the placeholder URL with the actual domain generated by Railway in Phase 2.
4. Commit this change and push to GitHub.
5. Vercel will automatically rebuild and deploy your frontend with the updated backend URL.

## Security Considerations

### API Keys
- Never commit your `.env` file to version control.
- Ensure your `GROQ_API_KEY` is set as an environment variable in the Railway dashboard.

## Troubleshooting

### Common Issues
1. **File Not Found Errors**: If the backend crashes on startup with a `FileNotFoundError` for the CSV dataset, ensure the `BASE_DIR` logic in `src/restaurant_app.py` is resolving correctly relative to where Railway runs the app.
2. **CORS Errors**: If the frontend cannot communicate with the backend, verify that `CORS(app)` is configured correctly in `src/restaurant_app.py` and that the `API_BASE` domain exactly matches the Railway domain.
3. **API Limitations**: Keep an eye on your Groq API limits to ensure requests are not throttled.
